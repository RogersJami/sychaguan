谢谢。我现在读大三，马上要考研复习了。也许是不熟悉的原因吧，从大一起对计算机没多大兴趣，也是浑浑噩 ...

这个你要先搞清楚it行业里面的分工关系，不考虑售前售后之类的服务性的职位，一般过程是公司决定一个方向，cto开始规划产品，然后开始由产品经理分析需求设计产品以及操作流程，然后项目经理组织开发，部分接项目的公司由于需求是客户提出的，项目经理可能和产品经理是一个人，项目经理决定开发进程控制以及接口设计，然后再由程序员开发出来，也就是所谓码农，有些公司这里颗粒做的细一些，项目经理设计到接口和输入输出定义，有些粗糙一些，直接程序员自己定某个功能模块下面的代码结构设计，至于为什么说程序员混不长呢，这是因为it行业发展太快，如果是应用层面的经验，没几年就更新掉了，比如过去用c的时代，那时候字符串编码处理都要自己写函数，现在直接用封装好的包就可以了，这些就是生产力进步和前人技术积累的体现，对应到语言也就是c c++ java等流行的不断变化，本身也反映了从电脑上程序到以浏览器为主流应用的体现，新的这些东西，学习周期也就一年多，开发效率取决于你对库函数的熟悉程度，自然老程序员比起来没啥优势，更何况这行当，待遇本身就有点虚高。
其实我觉得你也没什么可多想的，就像是制造业工人一样，高一层就是车间主任，再高一层就是经理之类的，你在分工中处于什么位置决定了你的收入，如果真的不适合干管理或者产品设计，那么也就是跟随父辈的脚步当一辈子工人而已，指望研究能发现这行当里面有什么稳赚不赔的点，是不现实的，任何行业都没有。

1、现在这个社会竞争肯定很激烈的，不存在说只是程序员竞争激烈。不过关键要有金刚钻，培训班的只有少部分人技术还过得去。
2、DBA，大型机系统的管理员没接触过，是否专业轻松就不清楚。当然oracle的DBA，10W年薪是完全有可能，不过对于应届生来说很难对oracle很熟悉，毕竟你不会接触到这么大型的数据库。

4、最后关于考研和就业，我个人倾向于本科毕业就去参加工作，在工作中学习积累经验。建议看一本书《系统程序员成长计划》，作者：李先静。在接下来的时间把一门语言学精通，并用它来实现你那些教材上的算法和实例。最后再强调一下：考研一定要考32所自主招生的院校或中科院那种机构。考一般的院校不如不考。
