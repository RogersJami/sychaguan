再道一声辛苦。

创意类产品的设计工作，我觉得最难的是沟通。可能需要一个素质较高的团队长时间的磨合 ...

其实创意什么的难度不大的，这个做书什么的，我的标准是很基本的，一般就可以，又不是艺术作品。所以技术不是主要的，这次主要还是工作方式的磨合，是工作思路冲突问题，比如我跟美编合作，习惯先说个大致的思路，让对方自己去递交作品，然后我再根据作品提出改进意见，多年来都是这么做的。但，现在的美编，根据我提的大致的标准，所递交的作品，让我没办法提改进意见和建议，我也不好意思提，因为，实在是相差太远了。于是返工……再返工……很后来才发现，对方的工作思路是，需要我做出特别细致特别明确的标准（那变成全部是我的设计和创意了），并习惯被全部否定，习惯返工，作品是在不断打击不断被否定中产生的，而我习惯了不断鼓励，在鼓励中让群众拿出符合要求的作品的，因材施教的味道比较足的。同时，对于美编的文字处理能力，再强的美编，我也是向来不放心的，必然亲自督工（盯着修改完），从开始就能把错误率降到最低，几乎不用怎么返工。但现在的美编，不喜欢别人督工，结果必然是各种返工……另外，做出版，不同的出版社，流程不同，风格节奏等更是差异很大。这些跟我已有的经验都有冲突，并由于我好歹也算是个业内资不浅人士，所以，别人还要尊重并适应我的风格，听取我的意见和建议，加上我很少有时间和精力顾及这块，加上人家也很忙，所以结果是各种“好事多磨”。不过还好，好歹都快完工了。

其实创意什么的难度不大的，这个做书什么的，我的标准是很基本的，一般就可以，又不是艺术作品。所以技术 ...

是啊，沟通的就是蓝图、目标、基本样式。双方的意见统一到一张设计图纸上面。

比如我做网站的思路，要考虑到老板的意图、目标，也要考虑后台技术资源的局限性。然后提出现实可行的路径。
由于项目开始是老板和技术宅男谈的，我花了3周时间才搞清楚他们两个人的想法其实不一样，而且老板本身根本没什么想法，只提了一个要求：大气、专业、美观。然后指着几个门户网站来让技术宅男“吸收灵感”。所以技术宅男搞的框架就是一个门户网站的样子。不仅不美观不大气而且很多东西根本没有资源来支撑。

所以争论几次的结果是按照我的老意见来，根据行业内成熟领先的公司网站来做。但是有因为成本有限（快一个月了老板才晓得成本资源有限会限制目标）也不能搞的太唯美。

现在的问题是老板越来越看我不顺眼，觉得我处处自以为是，“觉得自己是上帝”。还认为我的思路是剽窃的他的……实在是很晕。难道一般项目管理的常识很难理解吗？
