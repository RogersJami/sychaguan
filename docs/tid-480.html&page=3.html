不了解具体情况，不好全面分析。
有机会进入核心岗位当然好，但如果还没进入，不妨在已有的岗位上，把相关的东西学深学透。
比如黑盒测试，需要：
1、要保证测试的全面性、完备性------整体把握能力
2、考虑特殊情况、异常情况------反向思维能力
3、测试出问题，进行基本定位，别直接给开发人员，特别是复杂测试环境下测试出的问题------分析能力
一个好的测试人员，甚至应该具有需求分析的能力，从客户的角度考虑问题。

围绕岗位学习东西、锻炼能力，比简单的“接触的东西很多很广泛”要重要太多。
